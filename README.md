# Voxel Garden Viewer

Voxel Garden Viewer is a lightweight Expo SDK 55 app for previewing game models on iOS and Android with a touch-first Three.js scene, persistent on-device library management, and a built-in local LAN HTTP server/API.

## What it does

- Imports MagicaVoxel `.obj + .mtl` pairs from device file picker.
- Also imports `.gltf` and `.glb` files (single-file previews).
- Persists models and folders on-device across restarts.
- Lets you preview models with scene presets and lighting presets.
- Supports standard 3D gestures: orbit, pan, and pinch zoom.
- Shows a live debug HUD (FPS, draw calls, faces, vertices, meshes, materials).
- Includes in-app model library with recent list, rename/delete, and folder organization.
- Starts an automatic local web server at app launch with:
  - Browser upload/management UI built with Material UI.
  - OpenAPI spec at `/api/openapi.json`.
  - mDNS alias target: `http://model-viewer.local:18422` (with IP fallback when alias is unavailable).

## Setup on another machine

1. Install required tooling:
- Node 20+
- npm 10+
- Xcode + iOS simulator support (for iOS)
- Android Studio + SDK/emulator (for Android)

2. Clone and install:
- `git clone <repo-url>`
- `cd model-viewer`
- `npm install`

3. Generate native projects (required for native server and mDNS modules):
- `npx expo prebuild`

4. Run:
- iOS: `npm run ios`
- Android: `npm run android`

## Using the app

1. Open `Viewer` tab.
2. Tap import action and select an `.obj` and matching `.mtl`.
3. Toggle scene/lighting/debug settings from the settings action.
4. Switch models quickly from the model picker action.
5. Use `Library` tab for folders, recent models, rename/delete/move.

## Local web server and API

The server starts automatically at app launch.

Use `Server` tab to see:
- Alias URL: `http://model-viewer.local:18422`
- IP fallback URL (if alias is unavailable)
- API endpoint list

### Browser UI

Open the server URL in desktop/mobile browser on the same LAN.

- Upload screen supports drag-and-drop model files.
- Library screen supports rename/delete for models and folders.
- Changes reflect live in-app via polling and shared storage updates.

### OpenAPI

- `GET /api/openapi.json`

### API summary

- `GET /api/health`
- `GET /api/library`
- `POST /api/import-base64`
- `POST /api/folders`
- `PATCH /api/models/{modelId}`
- `POST /api/models/{modelId}` (iOS-compatible fallback for rename/move/select)
- `DELETE /api/models/{modelId}`
- `PATCH /api/folders/{folderId}`
- `POST /api/folders/{folderId}` (iOS-compatible fallback for rename)
- `DELETE /api/folders/{folderId}`

### Upload request example

```json
{
  "folderId": null,
  "files": [
    { "name": "castle.obj", "base64": "<base64-data>" },
    { "name": "castle.mtl", "base64": "<base64-data>" },
    { "name": "castle.png", "base64": "<base64-data>" }
  ]
}
```

When OBJ/MTL references texture files (`map_Kd`), include those image files in the same upload payload so materials resolve correctly on-device.

## Deploying to physical test devices

### iPhone

1. Connect device by USB.
2. Ensure Developer Mode is enabled.
3. Open Xcode project generated by prebuild (`ios/`), set signing team.
4. Run `npm run ios` with the device selected (or run from Xcode).

### Android

1. Enable Developer Options + USB debugging.
2. Confirm device via `adb devices`.
3. Run `npm run android`.

## Notes for future Codex instances

- This app intentionally uses native modules and therefore requires prebuild/dev-client workflows (not Expo Go).
- If mDNS alias fails due network policy, the app is expected to keep working via direct IP address.
- Local model data is stored under Expo document directory in `voxel-garden-viewer/`.
